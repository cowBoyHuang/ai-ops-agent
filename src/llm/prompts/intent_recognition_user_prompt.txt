用户问题: "{question}"

【打分标准】

每个意图从 4 个维度打分（0-1 分）：

1. 语义匹配度 (权重 0.5)
- 问题语义与意图的匹配程度
- 1.0 = 完全匹配，0.0 = 完全不匹配

2. 关键词匹配度 (权重 0.3)
- 是否包含意图相关的关键词
- 1.0 = 包含多个强相关关键词，0.0 = 无相关关键词

3. 上下文相关性 (权重 0.1)
- 是否符合对话上下文（如果没有上下文则为 0.5）
- 1.0 = 高度相关，0.0 = 完全不相关

4. 问题类型匹配 (权重 0.1)
- 问题类型是否符合意图特征
- 1.0 = 完全符合，0.0 = 完全不符合

【意图定义】

意图1: 系统逻辑咨询
- 定义: 咨询系统功能、代码逻辑、业务规则等
- 关键词: 支持、逻辑、规则、功能、如何、是否、代码、实现
- 问题类型: 咨询类（是否、如何、为什么）

意图2: 线上问题咨询
- 定义: 线上故障、错误、失败等问题排查
- 关键词: 失败、错误、问题、故障、排查、为什么不行、一直
- 问题类型: 描述问题/求助类

意图3: 订单信息查询
- 定义: 查询订单详情信息，如航班信息、乘机人信息、订单状态等
- 关键词: 查询、信息、详情、状态、航班、乘机人、订单号
- 问题类型: 查询类（什么、怎么样、在哪里）
